## [核心API概念](#core-api-concepts)

Vidau API在所有服务中遵循一致的设计模式。理解这些核心概念将帮助您有效地使用我们的所有端点。

### [任务基础架构](#task-based-architecture)

我们的API围绕**任务**概念构建。每个请求代表要执行的特定任务，如文本分析、图像识别或语音处理。这种任务基础架构实现了：

- 跨不同功能的一致请求结构
- 计算密集型操作的异步处理
- 基于任务需求的高效资源分配

每个API调用接受对象数组作为输入，其中每个对象代表要执行的单个**任务**。您可以在单个请求中发送任意数量的任务，每个任务都将独立处理。每个对象的结构根据任务类型而变化。

### [通用请求参数](#common-request-parameters)

每个API请求共享这些基本参数：

- taskType: 标识要执行的操作（textAnalysis, imageRecognition, speechProcessing）
- taskId: 您生成的唯一标识符，用于跟踪请求和匹配响应。如果任务失败或出现意外行为，在联系支持时提taskId能帮助我们快速定位请求并提供帮助
- includeCost: 可选标志，用于在响应中包含成本信息

```json

[

  {

    "taskType": "textAnalysis",

    "taskId": "a770f077-f413-47de-9dac-be0b26a35da6",

    "includeCost": true,

    // 任务特定参数...

  }

]
```
